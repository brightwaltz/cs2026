# HCI Trade-off Simulator

情報科学を学ぶ大学生向けの、**完全クライアントサイド**で動作するWeb学習アプリです。
セキュリティ警告UIを設計しながら、「防御力」と「利便性（UX）」の**トレードオフ**と、
その評価に使われる**内部アルゴリズム（計算式）**を批判的に学ぶことを目的としています。

外部APIやデータベースは一切使用せず、すべての処理がブラウザ内で完結します。

## 特徴

- **2タブ構成**
  - **入力・設計タブ** … シナリオ・警告文・ボタンの色／配置・強制待機タイマーなどを設計
  - **アルゴリズム解析タブ** … スコア計算の**実ソースコード**をシンタックスハイライト付きで表示（`?raw` 取り込みのため画面表示と実行コードが常に一致）
- **常時表示のプレビュー＆スコア** … 設計を変えると擬似警告UIと2つのスコアバーが即座に更新
- **レポート出力** … LMS 提出用テキストをワンクリックでクリップボードへコピー

## 評価アルゴリズム（`src/lib/scoring.ts`）

| 条件 | 防御力 | 利便性 |
| --- | --- | --- |
| ベース | 50 | 100 |
| テキスト100文字以上 | -20 | -15 |
| ボタンが赤 | +15 | — |
| ボタンが緑 | -20 | — |
| チェックボックス必須 | +20 | -20 |
| 強制待機タイマーあり | +25 | -40 |

いずれも 0〜100 にクランプされます。**この配点は「一つのモデル」であり唯一の正解ではありません。**

## 開発

```bash
npm install      # 依存関係のインストール
npm run dev      # 開発サーバ
npm run build    # 静的サイトを dist/ に出力
npm run preview  # ビルド結果のローカルプレビュー
```

`npm run build` だけで `dist/` に静的サイトが生成されます。複雑なルーティングは使用していません。

## GitHub Pages へのデプロイ

`vite.config.ts` で `base: './'` を指定しているため、リポジトリ名に依存せずプロジェクトページで動作します。

1. リポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に設定
2. `main` ブランチへ push すると `.github/workflows/deploy.yml` が自動でビルド＆デプロイ

手動で配置する場合は `npm run build` の出力 `dist/` をそのまま静的ホスティングに置くだけで動作します。
